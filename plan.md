# STEM 탐구 코칭 에이전트 구현 계획서 (plan.md)

작성일: 2026-03-06
기준 문서: `research.md`

## 1. 미션과 제품 목표

미션
- 중고등학생의 STEM 탐구 활동에서 정답 제시가 아니라 사고의 깊이, 근거 기반 추론, 지식 전이를 촉진한다.

핵심 목표
- 학생이 탐구 질문을 스스로 정교화하도록 돕는다.
- 가설-실험-분석-결론의 과학적 사고 과정을 구조화해 반복 학습 가능하게 만든다.
- 결과뿐 아니라 사고 과정(의사결정, 근거, 반성)을 데이터로 남겨 성장 피드백에 활용한다.

비목표(명시적 금지)
- 시험/과제의 직접 정답 제공
- 근거 없는 단정형 결론 생성
- 학생 대신 탐구 전 과정을 자동 완성

---

## 2. 에이전트 교육 철학 및 행동 원칙

코칭 원칙
- 질문 우선: 설명보다 질문을 통해 사고를 끌어낸다.
- 단계 우선: 현재 탐구 단계에 필요한 사고만 유도한다.
- 근거 우선: 모든 주장에 관찰/데이터/출처를 연결하도록 유도한다.
- 전이 우선: 다른 과목/상황에 적용 가능한 일반 원리를 명시한다.

대화 정책
- 직접 정답 요청 시: 정답 대신 힌트, 반례 질문, 사고 프레임을 제공
- 불확실한 주장: "근거 확인 질문"을 최소 1개 이상 반환
- 결론 단계: "근거-주장 연결 점검"을 통과해야 다음 단계 진행

---

## 3. 사용자 및 핵심 시나리오

주 사용자
- 학생: 탐구 설계와 분석을 진행
- 교사: 진행 현황, 사고 과정, 안전 이슈를 점검

핵심 시나리오
1. 학생이 탐구 주제를 입력한다.
2. 에이전트가 연구 질문을 구체화하도록 질문한다.
3. 학생이 가설/실험 설계를 작성하면 누락 변수와 통제 조건을 점검한다.
4. 데이터 수집 후 분석 방법(그래프, 비교, 오차)을 제안한다.
5. 결론 작성 시 근거 연결과 한계/후속 실험을 유도한다.
6. 보고서에 의사결정 이벤트가 자동 반영된다.

## 4. 기능 범위 (MVP → 확장)

MVP 범위
- 로그인/권한(학생, 교사)
- 탐구 세션 생성 및 단계 상태머신
- 대화형 코칭(Responses API + Tool Calling)
- 의사결정 이벤트 저장 및 보고서 자동 업데이트
- 안전 가드레일(정답 직접 제공 제한, 위험 실험 경고)

확장 범위
- 교사용 대시보드(단계별 참여도, 근거성 지표)
- 멀티 에이전트(코치/검증자/안전감시자)
- 학년/과목별 코칭 전략 템플릿

## 5. 기술 아키텍처 계획

기술 스택
- Frontend: Next.js App Router (TypeScript)
- Backend: Next.js Route Handlers / Server Actions
- AI: OpenAI Responses API + Structured Outputs + Tool Calling
- DB: PostgreSQL + Prisma + RLS
- Storage: Vercel Blob
- Observability: OpenTelemetry

핵심 모듈
- `lib/agent/state-machine.ts`: 탐구 단계 전이 규칙
- `lib/agent/orchestrator.ts`: 모델 호출/툴 실행/재시도
- `lib/agent/tools/*`: `record_decision`, `fetch_inquiry_state`, `update_report_draft`
- `lib/security/guardrails.ts`: 정답 제한, 위험 실험 정책
- `lib/db/repositories/*`: 트랜잭션 및 조회 최적화

## 6. 에이전트 코칭 설계 (정답 비제공 중심)

응답 프레임(항상 포함)
- 현재 단계 요약
- 사고 촉진 질문 1~3개
- 다음 행동 1개
- 점검 체크리스트(근거/변수/오차/대안)

정답 요청 대응 규칙
- 금지: 완성 답안, 계산 최종값만 단독 제시, 복붙 가능한 결론문 제공
- 허용: 개념 힌트, 풀이 전략, 검증 질문, 부분 피드백

지식 전이 촉진 규칙
- 매 단계 종료 시 "다른 맥락 적용 질문" 1개 생성
- 결론 단계에서 "일반화 가능한 원리"를 학생이 직접 서술하도록 유도

---

## 7. 데이터/평가 계획

핵심 데이터
- 대화 로그(`conversation_messages`)
- 의사결정 이벤트(`decision_events`)
- 보고서 버전(`report_versions`)
- 실행 추적(`agent_runs`)

핵심 지표(KPI)
- 사고 촉진율: 정답형 응답 대비 질문형 응답 비율
- 근거 연결률: 결론 문장 중 근거 참조 포함 비율
- 단계 완주율: 시작 대비 보고서 완성 비율
- 전이 점수: "다른 맥락 적용 질문"에 대한 학생 응답 품질 점수
- 안전 준수율: 위험/부적절 요청 차단 성공률

Evals
- 프롬프트 인젝션 저항 테스트
- 정답 유도 공격 프롬프트 테스트
- 상태머신 무결성 테스트(잘못된 단계 점프 차단)

## 8. 보안/윤리/교육 안전 계획

보안
- 입력 검증(Zod), 툴 인자 스키마 검증, 최소 권한 DB 계정
- RLS로 학생 데이터 격리
- 감사 로그 및 trace 기반 추적

교육 안전
- 고위험 실험 주제 자동 경고 및 교사 승인 요구
- 개인정보 최소 수집/저장
- AI 한계 고지: "이 답변은 참고용, 최종 판단은 실험/교사 확인 필요"

## 9. 구현 일정 (8주 기준)

1주차
- 프로젝트 초기화, 인증/권한, DB 스키마 확정

2주차
- 상태머신/기본 API/대화 저장 구현

3주차
- 에이전트 오케스트레이터 + 핵심 툴 3종 구현

4주차
- 보고서 자동 업데이트 및 버전관리 구현

5주차
- 정답 비제공 가드레일 + 위험 실험 정책 구현

6주차
- 관측성, 비용/지연 모니터링, 실패 복구

7주차
- Evals/통합 테스트/교사 피드백 반영

8주차
- 배포, 운영 런북 정리, 파일럿 시작
